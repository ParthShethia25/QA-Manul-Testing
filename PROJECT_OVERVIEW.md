# Project Overview – E-Commerce QA End-to-End

## Objective
To validate the quality of an e-commerce system by testing its:

- UI layer (web front-end using OpenCart demo)
- Service layer (REST APIs using public demo APIs)
- Data layer (simulated backend validation using SQL)

## Scope
- User registration, login, logout
- Product search and listing
- Cart and checkout flow
- Basic order validation

## Out of Scope
- Performance and load testing
- Security penetration testing
- Mobile app testing

## Layers Under Test

- **UI Layer** – OpenCart demo store
- **API Layer** – Public e-commerce style APIs (e.g., Fake Store, ReqRes)
- **Backend Layer** – Sample database tables and SQL-based validation
